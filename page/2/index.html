echarts设计的弊端日常干活时，发现像echarts、chartsJs这样早期的通过配置的方式组织信息渲染图表的可视化库有一个通用的设计缺陷： 配置和渲染分离。echarts的程序流程中，渲染图表的方法ecIns.setOption(option)需要一个option参数，它包含数据、图形样式、图形信息等全部渲染所需内容。配置信息和渲染环境数据之间相互阻隔，没办法在同一个渲染周期相互访问。这一点通常体现在配置图表过程中需要图表真实渲染的数据时，无法直接获取信息，比如坐标轴的比例尺，画布的实际宽高等。这种情况只能等待第一次渲染完成后，才能从图表实例获取并以此再次修改图表配置项。

git命令之git_remote
发表于2024-03-13
之前的github.io仓库里只有构建之后的产物，还需要一个仓库管理静态博客的源文件，所以要给本地仓库重新制定一个远程仓库。下面是用到的git_remote相关的命令。 常用命令查看当前远程仓库的地址1git remote -v 删除已有的远程仓库关联1git remote rm origin 这里的origin是默认的远程仓库的别名。 添加新的远程仓库的地址1git remote add origin https://xxxxxxxxx 这里再次使用origin作为别名， 也可以用别的名字。 将本地分支与远程分支关联起来1git push --set-upstream origin main 这里的意思是将本地的main分支和远程的origin下的默认分支关联起来。
